AI Brain

A living, learning system

The brain runs a continuous feedback loop on every request. Every resolved query is a training signal. Every anomaly sharpens the reflexes. Every upstream response is a reward or penalty.

Kalman Filter

Adaptive user count estimation. Tracks real concurrent users per minute with noise-aware prediction. Drives threshold scaling.

UCB Bandit

Upper Confidence Bound algorithm selects upstream resolvers. Balances exploration vs. exploitation. Learns per-colo optimal choice.

Markov Chain

Query sequence prediction. Learns domain co-occurrence patterns to pre-warm the cache before the next query arrives.

Welford Stats

Online mean/variance computation without storing history. Anomaly scoring uses dual Welford on count + delta simultaneously.

Weibull Survival

Models upstream reliability over time. Shape and scale parameters updated per upstream to predict failure probability.

Q-Learning RL

Reinforcement agent for upstream policy. Reward = fast clean response. Penalty = error, timeout, NX rate spike. Adapts routing dynamically.

Seasonal Rhythm

7-day × 24-hour traffic pattern matrix. Learns when your traffic peaks. Used for anomaly baseline and predictive scaling.

Bayesian Fusion

Combines DGA, phishing, and bot priors into a unified threat posterior. Updates per observation using Bayes' rule.

Brain Continuity

Full brain serialized to KV + D1 every 30 minutes. Survives worker restarts, redeployments, and isolate rotations with zero learning loss.

Authentication

Append to the path. That's it.

Append your token or master key as the last segment of the URL. Both work identically — same position, same format, same everything.

https://your-worker.workers.dev/dns-query/<token-or-masterkey>
https://your-worker.workers.dev/admin/<token-or-masterkey>
https://your-worker.workers.dev/api/intelligence/<token-or-masterkey>
https://your-worker.workers.dev/<token-or-masterkey>         ← same as /dns-query

Generated Token — normal use

Set DNS_TOKEN_SECRET. Generate time-limited tokens via the /admin dashboard UI or the API. Tokens are HMAC-SHA256 signed, valid ±2 hours, and path-scoped.

# Generate a token (using master key to auth)
curl "https://…/api/token/<masterkey>?path=/admin"
# → { "fullPath": "/admin/67f1a2b3<64-hex>" }

# Then use it exactly like a master key
curl "https://…/admin/67f1a2b3<64-hex-chars>"
curl "https://…/dns-query/67f1a2b3<64-hex-chars>?dns=…"

Master Key — fallback / emergency

Set DNS_MASTER_KEY to any string you choose. Use it the same way as a generated token — just append to the path. Never expires. Reach for it when a generated token has expired or isn't working.

wrangler secret put DNS_MASTER_KEY
# → enter: mysecretmasterkey

# Identical usage to a generated token
curl "https://…/admin/mysecretmasterkey"
curl "https://…/dns-query/mysecretmasterkey?dns=…"
curl "https://…/api/intelligence/mysecretmasterkey"
⚠️ Never expires — keep it secret. Rotate by updating the Wrangler secret and redeploying.

Generate a token locally (Node.js)

async function generateToken(secret, path) {
  const ts = (Date.now()/1000|0).toString(16).padStart(8, '0');
  const key = await crypto.subtle.importKey(
    'raw', new TextEncoder().encode(secret),
    { name: 'HMAC', hash: 'SHA-256' }, false, ['sign']
  );
  const sig = await crypto.subtle.sign('HMAC', key, new TextEncoder().encode(ts + path));
  const token = ts + [...new Uint8Array(sig)].map(b => b.toString(16).padStart(2, '0')).join('');
  console.log(`https://your-worker.workers.dev${path}/${token}`);
}

generateToken('your-DNS_TOKEN_SECRET', '/admin');
generateToken('your-DNS_TOKEN_SECRET', '/dns-query');
generateToken('your-DNS_TOKEN_SECRET', '/api');  // covers all /api/* routes
No secrets set? The worker runs in open-access mode — no auth required on any endpoint.

Deployment

Ship it to the edge

Runs on Cloudflare Workers free tier for low traffic. Scales globally to every Cloudflare PoP automatically.

Quick Deploy

# 1. Create KV namespace
wrangler kv:namespace create DNS_KV

# 2. Create D1 database
wrangler d1 create dns-worker-db

# 3. Run schema migrations
wrangler d1 execute dns-worker-db \
  --file=./schema.sql

# 4. Deploy
wrangler deploy

# 5. Set secrets
wrangler secret put DNS_TOKEN_SECRET
wrangler secret put DNS_CACHE_SECRET

Verify Deployment

# Test DoH resolution
curl "https://your.workers.dev/dns-query\
?dns=example.com&type=A" \
  -H "Accept: application/dns-json"

# Health check (JSON)
curl "https://your.workers.dev/" \
  -H "Accept: application/json"

# Test all upstreams
curl "https://your.workers.dev/\
test?dns=cloudflare.com&type=A"

# AI brain state
curl "https://your.workers.dev/api/ai" \
  # token is in the URL path, not a header

About

What is this, really?

"An experiment in pushing Cloudflare Workers to their absolute limit — packing a full AI-driven threat intelligence platform into a single JavaScript file with zero npm dependencies."

It started as a simple DNS-over-HTTPS proxy. A clean implementation of RFC 8484 that ran upstream queries in parallel and cached the results. That was version 1.

Then came the question: what if it learned which upstream was faster? That was the UCB bandit. Then: what if it could detect malicious domains? That was DGA. Then: what if it could predict the next query? That was Markov. Then: what if it survived restarts? That was brain continuity.

Hundreds of AI-assisted iterations later, it has a Kalman filter estimating concurrent users in real time, a Weibull survival model predicting upstream failure probability, a Q-learning agent adapting routing policy on the fly, and 40+ threat detection subsystems scoring every request before it ever touches an upstream resolver.

Every subsystem abbreviation — DGA, PCB, CRL, CREP, RPSF, URDM, CSC, CQTA — represents a real detection or optimization algorithm, running on every single DNS query, under a millisecond, at the edge closest to your users.
